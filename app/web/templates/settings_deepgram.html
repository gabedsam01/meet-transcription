Deepgram API key

Each user provides their own Deepgram API key. It is stored encrypted (like your Google token) and is required before a transcription can run — there is no deployment-wide fallback.

Per-user Deepgram key entry and encrypted storage are delivered by the auth/users branch (feat/auth-users-settings) and are not wired into this branch yet. Once integrated, this page lets you save your personal key, and Run once is blocked until a key is set.
